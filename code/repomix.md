# /code/repomix - Repository Audit Orchestrator

Comprehensive codebase analysis workflow using repomix + gemini for generating actionable improvement todos.

Execute repository audit for: $ARGUMENTS

## 🎯 Command Purpose
**Problem Solved**: Manual code audits are time-consuming and inconsistent
**Solution Provided**: Automated end-to-end audit workflow with AI-powered analysis
**User Benefit**: Actionable, prioritized improvement roadmap in 2-3 essential files

## 📦 Output Files (Consolidated)
1. **repomix-output.md** - Clean repository snapshot (intermediate, can be deleted)
2. **AUDIT.md** - Comprehensive audit with validated findings and actionable roadmap
3. **.repomixignore** - Optimized ignore patterns for future runs

## Pre-Flight Checks

**Required Tools**:
- [ ] `repomix` - Repository packing tool
- [ ] `gemini` - CLI for AI analysis
- [ ] `gemini-specialist` agent available

**Validation**:
```bash
# Check repomix installation
if ! command -v repomix &> /dev/null; then
  echo "❌ repomix not found"
  echo "   Install: npm install -g repomix"
  echo "   Or: brew install repomix"
  exit 1
fi

# Check gemini installation
if ! command -v gemini &> /dev/null; then
  echo "❌ gemini CLI not found"
  echo "   Install instructions: https://github.com/google/generative-ai-go"
  exit 1
fi

# Check gemini-specialist agent
if ! ls ~/.claude/agents/gemini-specialist.md &> /dev/null; then
  echo "⚠️ gemini-specialist agent not found"
  echo "   Analysis quality may be reduced"
fi

echo "✅ All required tools available"
```

## 🔄 Execution Workflow

### Phase 1: Setup & Repomix Execution (Consolidated)

**Objective**: Generate optimized .repomixignore and create clean repository snapshot

**ACTION**: Detect project type, configure ignore patterns, and execute repomix

```bash
echo "🔍 Phase 1: Analyzing project structure..."

# Parse target directory
TARGET_DIR="${ARGUMENTS:-$(pwd)}"
cd "$TARGET_DIR" || exit 1

echo "  Target: $TARGET_DIR"

# Detect project type
PROJECT_TYPE="unknown"
if [ -f "go.mod" ]; then
  PROJECT_TYPE="go"
  echo "  Detected: Go project"
elif [ -f "package.json" ]; then
  PROJECT_TYPE="javascript"
  echo "  Detected: JavaScript/TypeScript project"
elif [ -f "composer.json" ]; then
  PROJECT_TYPE="php"
  echo "  Detected: PHP project"
elif [ -f "requirements.txt" ] || [ -f "pyproject.toml" ]; then
  PROJECT_TYPE="python"
  echo "  Detected: Python project"
elif [ -f "Cargo.toml" ]; then
  PROJECT_TYPE="rust"
  echo "  Detected: Rust project"
fi

# Backup existing .repomixignore if present
if [ -f ".repomixignore" ]; then
  BACKUP_FILE=".repomixignore.backup.$(date +%Y%m%d_%H%M%S)"
  cp .repomixignore "$BACKUP_FILE"
  echo "  📦 Backed up existing .repomixignore to $BACKUP_FILE"
fi
```

### Step 1: Execute Gemini Analysis

```bash

Analyze this codebase for PRODUCTION READINESS across 4 critical dimensions:

**⚠️ CRITICAL CONSTRAINTS:**
- Focus on production hardening (caching, metrics, security, performance)
- Do NOT critique core infrastructure or rebuilding patterns
- Only flag issues with measurable production impact
- Assume code is runtime-complete unless proven otherwise

### 1. OBSERVABILITY (Production Visibility)
- Missing structured logging patterns
- No distributed tracing for request flows
- Missing health checks or readiness probes
- Inadequate error tracking and logging
- Score: /10

### 2. PERFORMANCE (Production Scale)
- Missing caching layers (Redis, in-memory, CDN)
- Database query optimization opportunities (indexes, N+1 queries)
- Resource pooling gaps (connections, workers, goroutines)
- Blocking I/O in hot paths
- Missing rate limiting or circuit breakers
- Score: /10

### 3. SECURITY (Production Hardening)
- Input validation gaps with production impact
- Missing authentication/authorization on critical paths
- Exposed secrets or credentials in config
- SQL injection or XSS vulnerabilities
- Missing CORS, CSRF protection for public endpoints
- Information leakage in error responses
- Score: /10

### 4. RELIABILITY (Production Stability)
- Missing graceful shutdown handling
- No retry logic with exponential backoff
- Inadequate timeout configurations
- Missing database migration safety (rollback, idempotency)
- No deployment health verification
- Configuration reload without restart
- Score: /10

## ⚠️ Error Handling & Recovery

### Common Issues

| Issue | Detection | Resolution |
|-------|-----------|------------|
| repomix not installed | `command -v repomix` fails | Install via npm or brew |
| gemini not available | `command -v gemini` fails | Provide installation guide |
| Output too large | File >5MB | Suggest subdirectory analysis |
| Gemini API failure | gemini command error | Retry with backoff, save progress |
| No findings | Empty validated-audit.md | Celebrate clean code, suggest maintenance |

### Rollback Procedure

```bash
# If audit generates poor results
cd "$TARGET_DIR"

# Restore original .repomixignore
if [ -f ".repomixignore.backup.*" ]; then
  mv .repomixignore.backup.* .repomixignore
fi

# Clean up generated files
rm -f repomix-output.md AUDIT.md audit-raw.md

echo "✅ Audit files cleaned up"
```
